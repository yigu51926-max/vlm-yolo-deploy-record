# 2026-06-15 至 2026-06-16 项目调试记录

## 一、阶段目标

本阶段主要围绕 VLM + YOLO 多模态识别流程展开，完成单图识别、单路视频检测和多路 RTSP 视频流检测的初步调试，为后续多路实时视觉理解系统做准备。

## 二、主要工作内容

### 1. 单图 YOLO + Qwen-VL 测试

使用 `yolo_qwen_image_demo.py` 对本地图片进行测试，尝试完成以下流程：

输入图片 → YOLO 目标检测 → Qwen-VL 图像理解 → 输出检测与描述结果

主要工作包括：

- 确认 Qwen-VL GGUF 模型路径；
- 测试 llama.cpp 本地推理；
- 切换测试图片；
- 验证 VLM 对图像内容的描述能力；
- 分析 YOLO 检测结果。

### 2. 单路视频 YOLO 检测

使用 `single_video_yolo.py` 对单个视频输入进行目标检测测试。

主要目标：

- 验证 OpenCV 能否正常读取视频；
- 检查 YOLO 是否能对视频帧进行检测；
- 为后续 RTSP 视频流接入做准备。

### 3. 多路 RTSP 视频流检测

使用 `multi_rtsp_yolo.py` 尝试进行多路 RTSP 视频流检测。

主要目标：

- 构造多路 RTSP 输入；
- 检查多路视频流能否同时读取；
- 测试 YOLO 对多路视频帧的检测能力；
- 初步评估多路实时检测的稳定性。

### 4. FFmpeg 推流测试

尝试使用 FFmpeg 将本地视频推送为 RTSP 视频流，为多路视频检测脚本提供输入源。

测试过程中出现 FFmpeg 推流退出的问题，需要继续检查 RTSP 服务、输入文件路径、端口占用和推流参数。

## 三、遇到的问题

### 问题 1：YOLO 检测结果不稳定

可能原因包括：

1. 测试图片或视频目标不明显；
2. YOLO 模型类别与测试场景不完全匹配；
3. 置信度阈值设置不合适；
4. 输入尺寸、帧率或图像质量影响检测效果。

后续解决思路：

- 更换道路车辆和监控场景视频；
- 降低置信度阈值；
- 保存检测结果图像进行人工确认；
- 对比不同 YOLO 权重的检测效果。

### 问题 2：多路 RTSP 推流不稳定

可能原因包括：

1. RTSP 服务未正常启动；
2. FFmpeg 推流地址或端口配置错误；
3. 视频源文件路径错误；
4. 多路推流时资源占用较高；
5. OpenCV 读取 RTSP 流存在延迟或断流。

后续解决思路：

- 先保证单路 RTSP 能稳定读取；
- 再逐步扩展到两路、多路；
- 使用日志记录每一路视频流状态；
- 对读取失败的视频流进行异常处理。
